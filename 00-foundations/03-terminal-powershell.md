# Terminal Powershell

## Главная идея в одном предложении
Работа с командной строкой, без графического интерфейса. Это быстро и помогает автоматизировать некоторые процессы. А так же это дает точность работы с файлами. 

## Что я узнал
1) Базовый синтаксис и начальное понимание того как работают "запросы". Verb - Noun, удобная логика, потому что, таким образом можно интуитивно понять, какой запрос можно дать в командную строку.

2) Основные команды которые я освоил на практике:
   - cd - где я сейчас
   - cd "folder_name" - перейти в папку
   - dir - вывод списка папок и файлов
   - mkdir - создание папки
   - ni - создание файла
   - rm - удаление файла
   - rm -Recurse "folder_name" - удаление всей папки (Благодаря рекурсии)
   - cat "file_name" - вывод содержимого файла
   - cp "file_name" "folder_name" - копия файла в папку
   - mv "file_name" "folder_name" - перемещение файла в папку
   - mv "file_name" "new_file_name" - переименование файла/папки

3) Алиасы — концепция, что ls, cd, cat — это не настоящие имена команд, а синонимы для Get-ChildItem, Set-Location, Get-Content.

4) Конвейер | — это способ соединить команды. Результат одной передаётся как вход следующей. В PowerShell конвейер передаёт объекты, а не строки текста — это его особенность по сравнению с обычным cmd.

5) Разница между "|" и ";".
   - "|" - это конвейер, одна единая команда одной строке.
   - ";" - этот символ означающий конец команды. Т.е если указать комада1; команда2; то это будет не конвеером, а 2 разными командами.

6) Рекурсия — «делать то же самое внутри того, что только что нашёл». В файлах: применить операцию к папке, потом к каждой папке внутри неё, и так далее.

## Полезное на будущее
- ls -Force        # показать также скрытые файлы
- ls -Recurse      # показать всё внутри, рекурсивно
- ls *.cs          # показать только файлы с расширением .cs
- ls -File         # только файлы
- ls -Directory    # только папки
